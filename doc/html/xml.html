This is a placeholder for a (future) document describing all the XML files that BOXER reads and writes.

Presently, BOXER can read and/or write the following data structures in XML format:

  • Data sets
  • Suites
  • Feature dictionaries
  • Learner description (just learner params, or a full model)
  • A "learner complex", which includes a feature dictionary, a suite, and one or more learners using this suite.

For an overview of major BOXER XML elements, see tags.html

Boxer uses Xerces to parse and generate XML.
