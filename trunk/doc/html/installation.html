Boxer and BORJ installation notes

Updated 2010-11-05

Prerequisites

JRE

Boxer has been compiled with Java 1.6, and that's what you need to have to use the precompiled Boxer binaries. More precisely, you'll need just the JRE to run Borj (or any other precompiled application that uses Boxer), or the full JDK to compile it. If you're writing your own Java code, then you must have the JDK already, of course.

A suitable version of JDK would be e.g. the one called Java SE 6 by Sun.com. It as available for download, for several platforms, from Java SE download site, where it's called "java Platform (JDK)".

As an alternative to downloading JRE/JDK from Sun.com, users of some some operating systems may obtain an implementation of JRE or JDK from the provider of their OS. For example, Ubuntu users can install JDK with 

  sudo apt-get install  openjdk-6-jdk

Naturally, if you already have full Java EE (which is JDK + some additional libraries and tools) it would do as well.

If you want to use Boxer with an earlier version of Java, you can also download the Boxer source code and try to re-compile it with your version.

Apache Ant

Apache Ant is needed only if you want to build Boxer and BORJ from source.

Third-party libraries

To compile and use Boxer and Borj, you need an external JAR file (or files):

  • xercesImpl.jar: There are multiple sources on the internet (including the official site, Apache Xerces), and if you've downloaded some of popular Java-based applications (e.g., Apache Ant), you probably already haver this file on. This library is essential for Boxer's parsing of XML files.

This is how to find xercesImpl.jar on the official Apache Xerces site: go to the binary download section of that site, http://xerces.apache.org/mirrors.cgi#binary, find the "Xerces2 Java" subsection, and download e.g. the zip file from that subsection. (As of June 2010, it is labeled "Xerces2 Java 2.9.1 - zip".) Once the zip file has been downloaded, unpack it (or, in MS Windows, "open" it using whatever program you use to "open" ZIP files), and extract from it xercesImpl.jar.

In more recent version of Xerces (e.g. 2.10.0), another JAR file, named xml-apis.jar, is also included into the Xerces' ZIP file. If this is the case, it is avisable that use that JAR file as well.

Once downloaded, you should place the Jar file(s) somewhere where it (or they) would be included in your classpath. This will be explained in more details below.

If you also plan to compile and use the Boxer's Blackbook adapter, you need the Blackbook jar files, and a number of third-party jar files, such those for Jena and Commons Logging; however, discussing this is outside of the scope of this document.

Downloading and using Boxer binaries

If you plan to use Boxer, but not to modify it, you only need to download the "Boxer binaries", i.e. a a Jar file containing compiled Boxer and BORJ code.

This jar file can be downloaded from the download section on our Google Code project. As of the summer 2010, the most recent version there is boxer-0.8.001.jar

Once downloaded, you can just add the Boxer jar, as well as the necessary thrid-party libraries to your classpath. For example, if you run BORJ (a BOXER application included into the BOXER distribution), the Linux/UNIX command line may look like this: 

 java -cp boxer-0.8.001.jar:xercesImpl.jar:xml-apis.jar edu.dimacs.mms.borj.Driver ...
If you have written your own application that uses the BOXER API, and your application's main class is called MyMainClass, then your command line may look like this: 
 java -cp   boxer-0.8.001.jar:xercesImpl.jar:xml-apis.jar MyMainClass

Note: If you're using MS Windows rather than Linux/UNIX, you should use semicolon (;) rather than colon (:) between the components of the classpath in the examples above.

Downloading and compiling from source

If you consider modifying Boxer code for your needs, or building your own Boxer application using BORJ as a sample, then you'll need to download Boxer source code, and to build from source. This is easy, thanks to Apache Ant.

An advantaging of downloading the source code zip file is that it also contains a number of example data files and the Javadoc HTML (i.e., what you are reading now).

Downloading from Google Code with SVN

You can get the most current Boxer/BORJ source code snapshot from the Google Code site for our project. To do this, you need to have a Subversion client installed; on most Linux machines, it comes as the pre-installed "svn" command; elsewhere, you may want to look on the web for the SVN application you like.

This is how to download the entire boxer-bayesian-regression source tree: 

  mkdir boxer
  cd boxer
  svn checkout https://boxer-bayesian-regression.googlecode.com/svn/trunk/ boxer-bayesian-regression
You may want to add your Google Code username to the above command, e.g. --username vmenkov

This will create on your host (under your current directory, i.e. boxer) a directory named boxer-bayesian-regression, with subidrectories named src, sample-data, etc.

Downloading a ZIP file

Alternatively, you can obtain Boxer source code in a single zip file. If you're a member of the MMS.DEVELOP project at Rutgers, you can download a Boxer zip file from the project page in the Sakai system there. You should also find a version (named on the pattern boxer-*.zip) at the download section on our Google Code project. Alternatively, you may be reading this because someone has sent you a zip file on a CD or by email.

After you've downloaded the zip file, create a directory named boxer-bayesian-regression, and unzip your zip file there: 

  mkdir boxer
  cd boxer
  mkdir boxer-bayesian-regression
  cd  boxer-bayesian-regression
  unzip  boxer-0_7_004.zip

Compiling

Place the Xerces jar file into the directory located at ../lib, relative to boxer-bayesian-regression. That is, if you have boxer/boxer-bayesian-regression, then the jar file should be placed into boxer/lib. (If you already have that jar file living elsewhere, and don't want to create an extra copy, you can just create a symbolic link; or you can modify the classpath specified in build.xml)

The directory boxer-bayesian-regression contains file build.xml with instructions for compiling. To use it, simply type, in the directory boxer-bayesian-regression, 

    ant compile
If no errors have been reported, you'll have your code compiled to 
../classes

You can also type 

    ant jar
to have a jar file, containing all the compiled code, built in the directory 
../lib

Using

Once the Boxer and Borj code has been compiled, you can use it in the same way as if you had downloaded the precompiled version (see above). Just make sure that the classes directory (or the new jar file, if you've built it) are in your classpath.

You can find more specific usage examples in the documentation for BORJ.
